When solving any leetcode question, we will follow the below strategy:
- We will read the problem, we will try to get the hints from the problem and then try map it to a known pattern (like binary search, DFS, BFS etc..). Also look at the constraints the problem
has been provided with. The constraints also provide hints on what complexity should the problem should be solved in.
- Use the framework to see if it solves the problem, write down the problem into the notes we are making so that I can identify and map better for the future problems.
- Then use the framwork to solve the problem.
- We are going to write rough algorithm to solve the problem. We will analyze the time and the space complexity here in this step.
- Then we will go over an example to check if the algorithm is actually going to solve the problem.
- Then we will go ahead and write the code, we will write clean code only.
- Then we will re-verify the written code by dry running the code with an example.
- Then we will run the code.
